##  Назначение проекта
Проект автоматизирует поиск контактных данных через внешний Telegram-бот и web-отчёт.
Основной сценарий работы:

- подать на вход `csv/xlsx` в `run_pipeline.py`
- или отправить `csv/xlsx` в Telegram через `tg_file_pipeline_bot.py`

Пайплайн поддерживает:

- поиск по `названию + ИНН`
- прямую сводку по номеру телефона в формате `7XXXXXXXXXX`

## Основные точки входа

- `run_pipeline.py` - основной оркестратор для запуска по входному `csv/xlsx`
- `tg_file_pipeline_bot.py` - Telegram file-bot, который принимает файл в чате и отправляет готовый `xlsx` обратно
- `qr_login.py` - первичная авторизация Telethon через QR и создание файла сессии
- `util_print_tg_chat_id.py` - получение `ID_TG_CHAT` для режима `tg_file_pipeline_bot.py`

## Остальные файлы

- `get_director_phone.py` - сценарий для ООО / АО / ЗАО через Telegram-кнопки
- `get_ip_phone.py` - сценарий для ИП через ссылку на web-отчёт
- `get_phone_summary.py` - сценарий для краткой сводки по номеру телефона
- `.env` - настройки Telegram и имя бота

## Установка

```bash
pip install -r requirements.txt
python -m playwright install
```

## Настройка `.env`

Минимум для запуска `qr_login.py`, `run_pipeline.py` и ручных сценариев:

```env
API_ID=123456
API_HASH=your_hash
SESSION_NAME=tg_user
BOT=@your_bot_username
```

Дополнительно для режима `tg_file_pipeline_bot.py`:

```env
TG_BOT_TOKEN=your_file_bot_token
ID_TG_CHAT=
```

Необязательные переменные можно добавить позже, если они действительно нужны:

```env
# уровень логов; по умолчанию INFO
LOG_LEVEL=INFO
# 0 - показывать окно браузера, 1 - работать скрыто; по умолчанию 1
PLAYWRIGHT_HEADLESS=0
# использовать ли прокси для Telethon и Telegram Bot API; по умолчанию false
USE_PROXY=false
# настройки прокси для Telethon и Telegram Bot API; используются только если USE_PROXY=true
PROXY_TYPE=socks5
PROXY_HOST=127.0.0.1
PROXY_PORT=10808
# пауза между шагами внутри обработки одной строки; по умолчанию 3
PIPELINE_STEP_DELAY_SECONDS=3
# пауза между строками входного файла; по умолчанию 5
PIPELINE_ROW_DELAY_SECONDS=5
# имя итогового csv-файла пайплайна; по умолчанию pipeline_results.csv
PIPELINE_RESULTS_CSV=pipeline_results.csv
# имя итогового xlsx-файла пайплайна; по умолчанию pipeline_results.xlsx
PIPELINE_RESULTS_XLSX=pipeline_results.xlsx
```

Если Telegram не открывается напрямую или `Telethon` / Bot API падают по таймауту, включи:

```env
USE_PROXY=true
```

и укажи рабочие `PROXY_TYPE`, `PROXY_HOST`, `PROXY_PORT`.

## Быстрый старт

### 1. Создать Telegram-сессию

```bash
python qr_login.py
```

Что делает этот скрипт:

- показывает QR-код в терминале
- ты сканируешь его в Telegram
- если включён 2FA, вводишь пароль Telegram
- после успешного входа создаётся файл `tg_user.session`

Если QR-код истёк, скрипт автоматически покажет новый.

### 2. Выбрать основной режим запуска

Если хочешь запускать пайплайн вручную по файлу:

```bash
python run_pipeline.py input.xlsx
```

Если хочешь отправлять файл в Telegram-группу и получать результат обратно:

```bash
python tg_file_pipeline_bot.py
```

Для Telegram-режима нужно заранее получить `ID_TG_CHAT`:

```bash
python util_print_tg_chat_id.py
```

## Ручные сценарии

Ниже описаны вспомогательные скрипты. Они полезны для отладки и точечных запросов, но для обычной работы по таблице основными остаются `run_pipeline.py` и `tg_file_pipeline_bot.py`.

### 1. Поиск телефона компании по ИНН

```bash
python get_director_phone.py
```

После запуска введи ИНН, например:

```text
5421110640
```

Скрипт:

1. отправит `/inn <ИНН>`
2. дождётся карточки компании
3. соберёт все кнопки на карточке
4. будет нажимать их по очереди сверху вниз
5. перед каждым нажатием подождёт 3 секунды
6. если попадёт в промежуточную карточку юрлица, продолжит обход дальше
7. когда дойдёт до карточки физлица, вытащит телефон и email
8. сохранит результат в `results.csv`

### 2. Поиск телефона для ИП

```bash
python get_ip_phone.py
```

Скрипт:

1. отправит `/inn <ИНН>`
2. дождётся ответа бота с кнопкой-ссылкой на полный отчёт
3. возьмёт URL из кнопки
4. откроет web-отчёт через Playwright
5. возьмёт первую личность сверху
6. вытащит телефон и email
7. сохранит результат в `results.csv` и `results.xlsx`

Если хочешь видеть браузер, можно добавить в `.env`:

```env
PLAYWRIGHT_HEADLESS=0
```

### 3. Краткая сводка по номеру телефона

```bash
python get_phone_summary.py
```

Скрипт:

1. отправит номер телефона боту
2. дождётся краткой сводки
3. вытащит основные поля
4. сохранит результат в `phone_summary_results.csv` и `phone_summary_results.xlsx`

Выход:

```text
/exit
```

## Если возникает ошибка авторизации

Если любой скрипт с `Telethon` пишет, что сессия не авторизована, нужно снова выполнить:

```bash
python qr_login.py
```

## Что сохраняют ручные сценарии

`get_director_phone.py` и `get_ip_phone.py` по умолчанию сохраняют результат в `results.csv` и `results.xlsx`.
Для этих файлов доступны необязательные переменные:

```env
RESULTS_CSV=results.csv
RESULTS_XLSX=results.xlsx
```

Основные поля результата:

- ИНН компании
- название компании
- ФИО директора
- ИНН директора
- телефон
- email
- ИНН физлица

## Единый пайплайн по таблице

Если есть входной `csv` или `xlsx`, можно использовать один из двух форматов:

- 1 столбец: название контрагента, 2 столбец: ИНН
- 1 столбец: номер телефона в формате `7XXXXXXXXXX`

можно запускать общий оркестратор:

```bash
python run_pipeline.py input.xlsx
```

Готовые примеры файлов лежат в папке `examples/`:

- `examples/input_company.xlsx` - пример для обычных компаний
- `examples/input_ip.xlsx` - пример для ИП
- `examples/input_phones.xlsx` - пример для списка телефонов

### Пример `xlsx` для компании

| Название контрагента | ИНН |
| --- | --- |
| ООО Ромашка | 7701234567 |
| АО Альфа | 7812345678 |

### Пример `xlsx` для ИП

| Название контрагента | ИНН |
| --- | --- |
| ИП Иванов Иван Иванович | 540123456789 |
| Индивидуальный предприниматель Петров Петр Петрович | 770123456789 |

### Пример `xlsx` для телефонов

Для этого режима лучше использовать один столбец без заголовка:

| Телефон |
| --- |
| 79001234567 |
| 79005554433 |
| 79998887766 |

Важно:

- номер должен быть строго в формате `7XXXXXXXXXX`
- `+7`, `8`, пробелы, скобки и дефисы не подойдут
- если отправляешь файл только с телефонами, лучше не добавлять заголовок
- файл с `названием + ИНН` может содержать и компании, и ИП в одном списке

Логика такая:

1. Если в первом столбце указан номер в формате `7XXXXXXXXXX`, сразу запускается `get_phone_summary.py`.
2. Иначе по названию определяется тип контрагента.
3. Если в названии есть `ИП` или `Индивидуальный предприниматель`, запускается сценарий `get_ip_phone.py`.
4. Иначе запускается сценарий `get_director_phone.py`.
5. Если телефон найден, автоматически запускается `get_phone_summary.py`.
6. Итог сохраняется в:
   - `pipeline_results.csv`
   - `pipeline_results.xlsx`

Можно переопределить файлы результата через переменные:

```env
PIPELINE_RESULTS_CSV=pipeline_results.csv
PIPELINE_RESULTS_XLSX=pipeline_results.xlsx
PIPELINE_STEP_DELAY_SECONDS=3
PIPELINE_ROW_DELAY_SECONDS=5
```

## Telegram-бот для файлов

Если нужен сценарий:

1. пользователь кидает `xlsx/csv` в Telegram-группу
2. бот принимает файл
3. запускает пайплайн
4. пишет статус обработки
5. отправляет обратно готовый `xlsx`

используй:

```bash
python tg_file_pipeline_bot.py
```

Нужно заполнить в `.env`:

```env
TG_BOT_TOKEN=...
ID_TG_CHAT=
```

Бот принимает только файлы из этого чата. Рабочие временные файлы складываются в `tg_bot_jobs/`.
Поддерживаются два варианта входного файла:

- таблица с `названием` и `ИНН`
- таблица со списком телефонов в первом столбце

Для режима с телефонами используй формат `7XXXXXXXXXX` без `+7`, пробелов, скобок и дефисов.
Для такого файла лучше не добавлять заголовок, а просто перечислить номера по одному в строке.
Готовые примеры можно взять из папки `examples/` и сразу отправить боту.
